# گردش کار گیت — نسخهٔ فارسی

قوانین و روند پیشنهادی:

- از شاخهٔ `main` برای کد پذیرفته‌شده استفاده کنید.
- برای هر ویژگی جدید یک شاخهٔ `feature/...` ایجاد کنید.
- قبل از کامیت، از `compile.sh` و تست‌های محلی استفاده کنید.
- یک pre-commit hook برای چک‌های سریع (فرمت، lint، verify.py) پیشنهاد شده است.

مراحل مرج:
1. باز کردن Pull Request از شاخهٔ feature
2. بررسی کد و مستندات
3. مرج پس از تایید
